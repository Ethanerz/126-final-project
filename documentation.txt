Both the signup and signing functions use the context functions in the AuthContext.jsx file so we'll start there
AuthContext.jsx:

signUpNewUser: 
- uses an in-built supabase function to create a new user using passed in arguments
- returns an error if the email is already in use, password doesn't follow the guidelines etc.
- returns success: true when supabase successfully creates a new user

signInUser:
- uses an in-built supabase function to sign in user using passed in arguments
- returns success: true when supabase successfully signs in the user

useEffect:
- sets the user's saved session to the current session e.g. keeps the user signed out whenever the user signs out until the user signs in again

signOut:
- uses in-built supabase function to sign out the user from the current session

Page directing/linking is handled using the router.jsx
router.jsx:
- uses react-router-dom functions to create links to other pages

Signup.jsx handles the signup functionalities
Signup.jsx:

handleSignUp
- calls signUpNewUser from AuthContext.jsx to try and create a new user account in supabase
- passes the saved email and password into the signUpNewUser function
- If user is successfully created (signUpNewUser returns success:true), it takes the user to the dashboard page (empty for now) using the react-router-dom function navigate()

Input Fields:
email field
- saves the value in the field into a temporary email variable whenever the field changes
password field
- saves the value in the field into a temporary password variable whenever the field changes

whenever the submit button is clicked, it runs handleSignUp

Signin.jsx:
- essentially the same as the signup page
- uses the signInUser instead to sign in the user based on the passed in arguments
- If user is successfully signed in (signInUser returns success:true), it takes the user to the dashboard page (empty for now) using the react-router-dom function navigate()
